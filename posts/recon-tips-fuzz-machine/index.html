Recon Tips - Fuzz Machine Setup

2021-10-20

If you are a security researcher, security engineer, red teamer, bug hunter or really anything related to security, you will, eventually, realize that it is next to impossible to keep everything mapped together in a company with a large infrastructure. Especially when needing to test from a blackbox approach.

There’s a lot of ground to cover, to keep servers secure it is necessary to have a minimum amount of automation running. Scripts scannning for takeovers, injections and fuzzing.

Fuzzing is not as easy as it seems, of course we can pick a wordlist from seclists and throw it into some tool and wait for results and filter from that. But, a lot of servers try to mitigate fuzzing atacks by returning spurious reponses sizes or 200 as status code even if the file does not exist. Not to mention it becomes increasingly harder to filter results for large scopes.

ffuf offers an auto calibrate solution that tries to bypass common fuzzing mitigrations. read about -ac and -acc flag in documentation.

The -ac flag helps a lot but running only once against a big wordlist is not the best approach, eventually you will notice that the application returns a lot of dull responses for paths that are not used by -ac.
